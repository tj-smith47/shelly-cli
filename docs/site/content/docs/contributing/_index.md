## Code of Conduct

We expect all contributors to follow our code of conduct. Be respectful, inclusive, and constructive.

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes following our coding standards
4. Write or update tests as needed
5. Run `go test ./...` and `golangci-lint run`
6. Commit with descriptive messages
7. Push and open a pull request

## Reporting Issues

Found a bug or have a feature request? Please open an issue on [GitHub](https://github.com/tj-smith47/shelly-cli/issues).
